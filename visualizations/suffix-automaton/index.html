Build a Suffix Automaton one character at a time.

The Suffix Automaton (SAM) is the smallest DFA that accepts every suffix of a string. Each state represents an endpos equivalence class — substrings that always end at exactly the same set of positions. Online construction is O(n) with at most 2n−1 states and 3n−4 transitions.

Online Construction Algorithm

For each new character c, create state cur with len = last.len + 1. Walk up suffix links from last, adding transition c → cur wherever it is missing.

When a state p already has c → q:

  • If q.len = p.len + 1 → set cur.link = q (no clone needed)
  • Otherwise → clone q into a new state, fix suffix links and transitions, set both cur.link and q.link to the clone

Endpos Equivalence Classes

Each SAM state represents a group of substrings with the identical endpos set — the positions in the original string where each substring ends.

The number of distinct substrings represented by state v is exactly v.len − v.link.len. Sum over all states to count all distinct substrings.

Why SAM is Powerful

  • Accepts all suffixes in O(n) states — provably optimal
  • Count distinct substrings: sum of len − link.len over all states
  • Find longest common substring of two strings in O(n + m)
  • Foundation for palindrome counting and compressed text indexes
