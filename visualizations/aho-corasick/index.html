How Aho-Corasick Works

1. Build the Trie

Insert all patterns into a trie (prefix tree). Each node represents a prefix of one or more patterns. Nodes that terminate a pattern are marked as output nodes.

2. Compute Failure Links

Using BFS, compute a fail link for each node: the longest proper suffix of the node's string that is also a prefix in the trie. The root's children all fail to root.

3. Compute Output Links

Also via BFS, compute output links (dictionary links): if a node's fail target has output, we chain that output here. This lets us find all overlapping matches at a position in O(matches) time.

4. Search

Scan the text left-to-right. At each character, follow trie edges. If no edge exists, follow failure links until we find one (or reach root). Any time we're at or pass through an output node, report a match. Total time: O(n + m + z) where z = match count.
