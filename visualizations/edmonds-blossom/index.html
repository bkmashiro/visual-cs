How It Works

Edmonds' Blossom Algorithm finds a maximum matching in a general (non-bipartite) graph. A matching is a set of edges where no two edges share a vertex. A maximum matching has the largest possible number of edges.

  • Augmenting path — an alternating path (unmatched → matched → unmatched → …) that starts and ends at unmatched vertices. Flipping matched/unmatched along it increases matching size by 1.
  • BFS tree — from each free (unmatched) vertex, build an alternating BFS tree to search for an augmenting path. Even-depth nodes are "outer", odd-depth are "inner".
  • Blossom detection — when two outer nodes from the same BFS tree are connected by an edge, this creates an odd cycle called a blossom. The blossom's base is the LCA (lowest common ancestor) of the two endpoints in the tree.
  • Blossom contraction — contract the blossom into a single super-node, then continue searching. If an augmenting path is found in the contracted graph, it can be lifted back to the original graph.
  • Augmentation — once found, flip all matched/unmatched edges along the augmenting path. Repeat until no augmenting path exists.
