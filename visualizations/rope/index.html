How It Works

Build leaves of size k, then merge adjacent nodes upward into a balanced binary tree. Internal node weight equals the length of its left subtree, so index lookup compares against that weight at every level. Split and concat become tree operations instead of repeatedly copying a full string.

Average operations are near O(log n) on a balanced rope, while the logical string stays the same as leaf1 + leaf2 + ....
