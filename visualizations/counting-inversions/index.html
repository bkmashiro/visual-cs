Current Operation

Generate an array and run modified merge sort to count inversions.

Pseudocode

countInversions(arr):
  if len(arr) ≤ 1: return 0
  mid = len(arr) / 2
  left = countInversions(arr[..mid])
  right = countInversions(arr[mid..])
  split = mergeCount(arr, mid)
  return left + right + split
// In merge: when right[j] < left[i],
// inversions += remaining in left half

How It Works

Counting inversions via modified merge sort runs in O(n log n). An inversion is a pair (i,j) where i < j but arr[i] > arr[j].

During the merge step, when an element from the right half is placed before remaining left elements, all those left elements form inversions with it. This is the key insight.
