Arithmetic Coding

Compress a message into a single real number in [0, 1). Each symbol narrows the interval proportionally to its probability — high-frequency symbols shrink it less, yielding fewer bits. The decoder recovers symbols by repeatedly finding which sub-interval contains the encoded value.

How It Works

1
Count frequencies of each symbol in the message.
2
Assign intervals [cumLow, cumHigh) proportional to probability.
3
Encode: start with [0,1). For each symbol, zoom into its sub-interval.
4
Output any value in the final interval (e.g. midpoint).
5
Decode: find which sub-interval the value falls in, emit symbol, rescale.

Complexity

Encode timeO(n · |Σ|)
Decode timeO(n · |Σ|)
Precision neededO(n) bits
Code length≈ H(X) + 2

Key Properties

📐
Near-optimal — achieves within 2 bits of the entropy lower bound.
🔢
Single output value — unlike Huffman which outputs per-symbol codes.
⚡
Adaptive variant — probabilities can update as symbols are seen.
⚠️
Precision — practical implementations use integer interval rescaling to avoid floating-point underflow.

vs. Huffman Coding

✅
Arithmetic coding handles non-power-of-2 probabilities without wasting bits.
✅
Better compression for highly skewed distributions (e.g. 99%/1%).
⚖️
Huffman is simpler and patent-free; arithmetic coding requires integer scaling tricks.
