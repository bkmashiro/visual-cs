Huffman Coding

An optimal prefix-free compression algorithm that assigns shorter bit strings to more frequent symbols. Build the Huffman tree step-by-step, inspect codes, and see live compression statistics.

Algorithm Steps

1
Count frequencies of each symbol in the input.
2
Create leaf nodes for each symbol and insert into a min-heap.
3
Extract two minimums, merge into an internal node with combined frequency.
4
Repeat until one node remains — the Huffman root.
5
Assign codes: traverse left → 0, right → 1.

Complexity

Build treeO(n log n)
EncodeO(m)
SpaceO(n)
n = unique charsm = input length

Key Properties

🌿
Prefix-free — no code is a prefix of another; decoding is unambiguous.
⚖️
Optimal — minimizes expected code length for a given symbol frequency distribution.
🔢
Greedy — always merges the two least-frequent nodes first.
📦
Lossless — the original data can be fully recovered from the encoded bitstream.
