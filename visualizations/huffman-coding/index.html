Why Huffman Works

  • The two smallest frequencies can safely become siblings in some optimal tree.
  • Merging them reduces the problem to a smaller instance with one combined weight.
  • Repeating this greedy choice yields an optimal prefix-free code.

Complexity

  • Frequency counting: O(n)
  • Heap-based tree build: O(k log k) for k distinct characters
  • DFS code generation: O(k)

Notes

  • For a single unique character, the visualization assigns code 0.
  • Displayed bit count uses the final code table even while you scrub earlier stages.
  • Character labels normalize spaces and punctuation so duplicates stay easy to track.
