How the Hungarian Algorithm Works

The Hungarian Algorithm (Kuhn–Munkres, 1955) solves the assignment problem in O(n³) time: given an n×n cost matrix, find a perfect matching between rows and columns that minimises the total cost.

  • Row reduction — subtract each row's minimum value. This shifts costs without changing the optimal structure, creating at least one zero per row.
  • Column reduction — subtract each column's minimum value. Creates at least one zero per column.
  • Line covering — find the minimum number of horizontal/vertical lines that cover all zeros (König's theorem: this equals the size of the maximum bipartite matching in the zero-pattern graph).
  • Optimality check — if the number of covering lines equals n, a complete zero-based assignment exists → extract it and stop.
  • Matrix adjustment — otherwise, let θ = minimum uncovered value. Subtract θ from all uncovered cells; add θ to double-covered cells. This creates new zeros without destroying existing ones, then repeat from step 3.
  • Assignment extraction — once n lines suffice, use augmenting paths on the zero-graph (bipartite matching) to extract the optimal assignment.

The algorithm is guaranteed to converge in at most n adjustment rounds. The total cost of the optimal assignment equals the sum of all row and column reductions applied plus the sum of the values in the original matrix at the assigned positions.
