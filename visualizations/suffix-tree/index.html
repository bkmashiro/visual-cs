Build a Suffix Tree one character at a time.

A suffix tree is a compressed trie of all suffixes of a string. Ukkonen's algorithm builds it online in O(n) time using three tricks: implicit extensions, the active point, and suffix links. The compact structure enables O(m) pattern search and many O(n) string algorithms.

Ukkonen's Algorithm

Process each character s[i]. Maintain three state variables:

  • activeNode — where we start matching from
  • activeEdge — first char of the edge being followed
  • activeLength — how far along that edge

The remainder counts pending suffixes not yet made explicit. When we can absorb a char implicitly (rule 3), we stop — all remaining suffixes exist implicitly.

Suffix links let us jump from a newly created internal node to the next shorter suffix in O(1), enabling linear total work.

Three Extension Rules

  • Rule 1 — Path ends at a leaf: extend the leaf edge label (free via open-ended leaves).
  • Rule 2 — No continuation for current char: create a new leaf (and possibly a new internal node).
  • Rule 3 — Current char already on an edge: do nothing — the suffix exists implicitly. Stop inner loop.

Why Suffix Trees?

  • Pattern matching in O(m) time (after O(n) build)
  • Longest repeated substring in O(n)
  • Longest common substring of two strings in O(n+m)
  • Count distinct substrings by summing edge lengths
  • Foundation for BWT and FM-index
