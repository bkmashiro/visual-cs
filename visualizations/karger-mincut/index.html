How It Works

Karger's Algorithm finds a minimum cut in an undirected graph — the smallest set of edges whose removal disconnects the graph into two parts. It uses randomized edge contraction.

  • Edge selection — pick a random edge (u, v) from the current multigraph.
  • Contraction — merge nodes u and v into a single supernode. The supernode inherits all edges from both nodes.
  • Self-loop removal — edges between u and v (now within the same supernode) become self-loops and are removed.
  • Multi-edges — edges to the same external node from both u and v are kept as parallel edges (they are not merged — this is crucial for correctness).
  • Repeat — continue contracting until only 2 supernodes remain. The remaining edges between them form a cut.
  • Probability — the probability that a specific min-cut of size k is found is at least 2/(n(n-1)). Running the algorithm O(n² log n) times gives high probability of finding the true min cut.
